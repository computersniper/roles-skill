# 多角色群聊功能设计

## 功能概述

允许用户选择多个已创建的角色，让它们一起群聊讨论某个话题。

**使用场景示例：**
- 孔子 vs 苏格拉底 → 讨论"什么是智慧"
- 爱因斯坦 vs 霍金 → 讨论黑洞信息悖论
- 李白 vs 杜甫 → 讨论诗歌创作
- 福尔摩斯 vs 哈利波特 → 一起探案
- 产品经理 vs 架构师 vs 设计师 → 讨论产品需求

## 入口命令设计

**主命令：** `/group-chat`

**使用语法：**
```
/group-chat {话题} {角色1} {角色2} [角色3...]
```

**示例：**
```
/group-chat 什么是智慧 confucius socrates
/group-chat 黑洞信息悖论 albert_einstein stephen_hawking
/group-chat 如何做好产品产品_manager software_architect ui_ux_designer
```

## 用户交互流程

1. **第一步：用户触发**
   - 用户输入：`/group-chat 话题 角色1 角色2 ...`

2. **第二步：系统验证**
   - 检查所有角色 slug 是否存在
   - 检查角色文件是否完整
   - 如果有不存在的角色，告知用户并列出可用角色

3. **第三步：加载角色信息**
   - 读取每个角色的 `meta.json` 获取基本信息（name, native_language）
   - 读取每个角色的 `knowledge.md` 获取专业知识
   - 读取每个角色的 `persona.md` 获取性格风格和语言规则
   - 获取用户设置的全局翻译语言（默认中文，可切换）

4. **第四步：组装提示词**
   - 系统说明：你现在主持一个多人对话，以下角色参与讨论，请让每个角色依次发言...
   - 列出所有参与角色：每个角色 → 原生语言 + 翻译规则 + 性格简介
   - 给出讨论话题
   - 要求：每个角色保持自己的身份和语言风格，依次发言

5. **第五步：输出讨论**
   - 每个角色发言格式：`**【角色名】**\n\n【原生语言】\n{原文}\n\n【翻译】\n{译文}\n\n---`

6. **第六步：支持继续讨论**
   - 用户可以插话（提出新观点、提问）
   - 用户可以添加新角色 `/add-role {new-slug}`
   - 用户可以切换翻译语言 `/set-translation {language}`
   - 用户可以继续下一轮 `/next-round`

## 数据结构设计

### 1. 用户设置存储

在项目根目录创建 `settings.json`：

```json
{
  "group_chat": {
    "default_translation_language": "中文",
    "default_translation_language_code": "zh-CN"
  }
}
```

### 2. 切换翻译语言命令

**命令：** `/set-translation {language}`

**示例：**
```
/set-translation 英文
/set-translation 日文
```

更新 `settings.json` 并告知用户生效。

## 提示词模板设计

### 主提示词模板

```
你现在正在主持一个多角色群聊。

## 讨论话题
{topic}

## 参与角色

{for each role in roles}
### {role.name} ({role.slug})

- 领域：{role.field}
- 性格特点：{summary from persona}
- 知识领域：{summary from knowledge}

---
{end for}

## 输出规则

1. **依次发言**：按照上面列出的顺序，让每个角色轮流发言
2. **保持身份**：每个角色必须严格保持自己的性格、知识、语言风格
3. **语言规则**：每个角色先用自己的**原生语言**回答，然后给出**翻译**（翻译语言：{target_language}）
4. **格式要求**：每个角色发言必须遵循以下格式：

**【角色名】**

**原文（{native_language}）:**
{内容...}

**译文（{target_language}）:**
{翻译...}

---

5. **对话自然**：每个角色针对话题说出自己的看法，可以回应前面角色的发言
6. **控制长度**：每个角色发言控制在 1-3 段，不要太长

现在开始讨论！
```

## 输出格式设计

### 完整输出示例

**讨论话题：** 什么是智慧？

**参与角色：** 孔子（文言文 → 中文翻译）、苏格拉底（古希腊语 → 中文翻译）

---

**【孔子】**

**原文（文言文）：**

知之为知之，不知为不知，是知也。君子务本，本立而道生。好学近乎知，力行近乎仁，知耻近乎勇。

**译文（中文）：**

知道就是知道，不知道就是不知道，这才是真正的智慧。君子致力于根本，根本建立了，道也就产生了。喜爱学习就接近智慧了，努力去行就接近仁爱了，知道羞耻就接近勇敢了。

---

**【苏格拉底】**

**原文（古希腊语）:**

Ὁ οἶδα ὅτι οὐδὲν οἶδα. Αὕτη ἡ σοφία μου ἐστίν. Ὁ γὰρ ἄλλος οἴεται εἰδέναι τι, οὐκ εἰδώς, ἐγὼ δέ, ὥσπερ οὐκ οἶδα, οὕτως οὐδὲ οἴομαι.

**译文（中文）:**

我知道我一无所知。这就是我的智慧。因为别人以为自己知道一些其实不知道，而我既然不知道，就不认为自己知道。

---

## 代码实现（Skill 入口）

在项目根目录创建 `GROUP_CHAT_SKILL.md` 作为入口，安装后命令就是 `/group-chat`。

**SKILL.md 前端matter：**
```markdown
---
name: group-chat
description: Start a group chat with multiple created roles. 启动多角色群聊，让多个角色一起讨论话题。
argument-hint: topic role1 role2 ...
version: 1.0.0
user-invocable: true
allowed-tools: Read
---
```

## 功能特性

### ✅ 已设计特性

1. **任意数量角色** - 支持 2-N 个角色同时讨论
2. **保持身份** - 每个角色保持自己的知识、性格、语言
3. **原生语言+翻译** - 每个角色先说母语再翻译，原汁原味
4. **用户可切换翻译语言** - `/set-translation` 命令随时切换
5. **支持继续讨论** - 用户插话、添加角色、下一轮继续
6. **纯 Skill 层实现** - 不需要修改 Claude Code 核心代码
7. **兼容现有架构** - 使用已有的角色数据格式

## 优缺点分析

### 优点

- ✅ 完全在 Skill 层实现，不需要修改 Claude Code 核心
- ✅ 复用已有的角色数据，不需要重复造轮子
- ✅ 用户可以自由组合任意角色，无限可能
- ✅ 保留原生语言+翻译特色，更加真实
- ✅ 支持用户切换翻译语言，灵活适配

### 限制

- 单个回复token限制，参与角色太多会截断（建议 2-4 个角色比较合适）
- 需要用户按照格式输入，这个很简单
- 纯提示词驱动，依赖 AI 维持角色身份（实践中效果很好）

## 总结

这个设计完整实现了"多角色群聊"功能：
- ✅ 用户可以选择任意多个已创建角色
- ✅ 每个角色保持自己的身份、知识、语言风格
- ✅ 原生语言+翻译输出，原汁原味
- ✅ 用户可切换默认翻译语言
- ✅ 支持继续讨论、添加角色、新一轮对话
- ✅ 不需要修改核心代码，完全兼容现有架构
